# 学习成果展示网站数据库设计文档

## 1. 数据库概述

### 1.1 数据库信息
- **数据库类型**：MySQL 8.0+
- **字符集**：utf8mb4
- **排序规则**：utf8mb4_unicode_ci
- **存储引擎**：InnoDB

### 1.2 设计原则
- 遵循数据库设计三范式
- 合理设置索引，优化查询性能
- 使用外键约束保证数据完整性
- 预留扩展字段，支持后续功能扩展
- 时间字段统一使用 DATETIME 类型

## 3. 数据表详细设计

### 3.1 用户表 (users)

存储用户基本信息，包括管理员和访客。

| 字段名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | 是 | 自增 | 主键ID |
| username | VARCHAR(50) | 是 | - | 用户名，唯一 |
| password | VARCHAR(255) | 是 | - | 密码（加密存储） |
| email | VARCHAR(100) | 是 | - | 邮箱，唯一 |
| nickname | VARCHAR(50) | 否 | NULL | 昵称 |
| avatar | VARCHAR(255) | 否 | NULL | 头像URL |
| bio | TEXT | 否 | NULL | 个人简介 |
| role | TINYINT | 是 | 0 | 角色：0=访客，1=管理员 |
| status | TINYINT | 是 | 1 | 状态：0=禁用，1=正常 |
| github_url | VARCHAR(255) | 否 | NULL | GitHub链接 |
| website_url | VARCHAR(255) | 否 | NULL | 个人网站链接 |
| created_at | DATETIME | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| last_login_at | DATETIME | 否 | NULL | 最后登录时间 |

**索引设计：**
```sql
PRIMARY KEY (id)
UNIQUE INDEX idx_username (username)
UNIQUE INDEX idx_email (email)
INDEX idx_role (role)
INDEX idx_status (status)
```

## 4. 外键约束

```sql
-- 项目表外键
ALTER TABLE projects
    ADD CONSTRAINT fk_project_category FOREIGN KEY (category_id) REFERENCES categories(id),
    ADD CONSTRAINT fk_project_author FOREIGN KEY (author_id) REFERENCES users(id);

-- 文章表外键
ALTER TABLE articles
    ADD CONSTRAINT fk_article_category FOREIGN KEY (category_id) REFERENCES categories(id),
    ADD CONSTRAINT fk_article_author FOREIGN KEY (author_id) REFERENCES users(id);

-- 笔记表外键
ALTER TABLE notes
    ADD CONSTRAINT fk_note_category FOREIGN KEY (category_id) REFERENCES categories(id),
    ADD CONSTRAINT fk_note_author FOREIGN KEY (author_id) REFERENCES users(id),
    ADD CONSTRAINT fk_note_parent FOREIGN KEY (parent_id) REFERENCES notes(id);

-- 分类表外键
ALTER TABLE categories
    ADD CONSTRAINT fk_category_parent FOREIGN KEY (parent_id) REFERENCES categories(id);

-- 项目标签关联表外键
ALTER TABLE project_tags
    ADD CONSTRAINT fk_pt_project FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    ADD CONSTRAINT fk_pt_tag FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE;

-- 文章标签关联表外键
ALTER TABLE article_tags
    ADD CONSTRAINT fk_at_article FOREIGN KEY (article_id) REFERENCES articles(id) ON DELETE CASCADE,
    ADD CONSTRAINT fk_at_tag FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE;
```

## 7. 性能优化建议

### 7.1 缓存策略

| 缓存内容 | 缓存时间 | 缓存位置 |
|----------|----------|----------|
| 首页数据 | 5分钟 | Redis |
| 文章详情 | 10分钟 | Redis |
| 分类列表 | 30分钟 | Redis |
| 标签列表 | 30分钟 | Redis |
| 访问计数 | 实时累加Redis，定时同步MySQL | Redis |

### 7.2 分页查询优化

对于数据量较大的表（文章、评论、访问日志），建议：
- 使用游标分页而非OFFSET分页
- 添加适当的索引覆盖
- 考虑数据归档策略

### 7.3 统计字段更新策略

点赞数、浏览数等统计字段建议：
- 使用Redis实时计数
- 定时批量同步到MySQL
- 减少频繁UPDATE操作
