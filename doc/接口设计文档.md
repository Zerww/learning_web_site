# 学习成果展示网站接口设计文档

## 1. 接口概述

### 1.1 基本信息
- **接口协议**：HTTPS
- **接口风格**：RESTful API
- **数据格式**：JSON
- **字符编码**：UTF-8
- **API版本**：v1
- **基础路径**：`/api/v1`

### 1.2 技术框架
- **后端框架**：Django 5.0 + Django REST Framework 3.14
- **认证方式**：JWT (JSON Web Token)
- **跨域支持**：CORS

---

## 2. 接口规范

### 2.1 URL命名规范

| 规则 | 示例 |
|------|------|
| 使用小写字母 | `/api/v1/articles` |
| 使用连字符分隔单词 | `/api/v1/user-profiles` |
| 使用复数形式表示资源集合 | `/api/v1/projects` |
| 使用资源ID定位单个资源 | `/api/v1/articles/{id}` |
| 子资源通过嵌套表示 | `/api/v1/articles/{id}/comments` |

### 2.2 HTTP方法语义

| 方法 | 语义 | 示例 | 是否需要认证 |
|------|------|------|--------------|
| GET | 获取资源 | `GET /articles` 获取文章列表 | 否（部分接口需要） |
| POST | 创建资源 | `POST /articles` 创建文章 | 是 |
| PUT | 更新资源（完整） | `PUT /articles/{id}` 更新文章 | 是 |
| PATCH | 更新资源（部分） | `PATCH /articles/{id}` 部分更新文章 | 是 |
| DELETE | 删除资源 | `DELETE /articles/{id}` 删除文章 | 是 |

### 2.3 请求头规范

| 请求头 | 说明 | 是否必填 | 示例 |
|--------|------|----------|------|
| Content-Type | 请求内容类型 | 是（POST/PUT/PATCH） | `application/json` |
| Authorization | JWT认证令牌 | 是（需认证的接口） | `Bearer {token}` |
| Accept | 期望的响应类型 | 否 | `application/json` |
| X-Request-ID | 请求唯一标识 | 否 | 用于追踪和日志 |
| Accept-Language | 语言偏好 | 否 | `zh-CN` |

### 2.4 响应格式规范

#### 2.4.1 成功响应格式

**单条数据响应：**
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "id": 1,
        "title": "文章标题",
        "content": "文章内容",
        "created_at": "2026-05-05T10:30:00Z"
    }
}
```

**列表数据响应：**
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "list": [
            {"id": 1, "title": "文章1"},
            {"id": 2, "title": "文章2"}
        ],
        "pagination": {
            "page": 1,
            "page_size": 10,
            "total": 100,
            "total_pages": 10,
            "has_next": true,
            "has_prev": false
        }
    }
}
```

**操作成功响应（无数据）：**
```json
{
    "code": 200,
    "message": "操作成功"
}
```

#### 2.4.2 错误响应格式

```json
{
    "code": 400,
    "message": "请求参数错误",
    "errors": [
        {
            "field": "title",
            "message": "标题不能为空"
        },
        {
            "field": "content",
            "message": "内容长度不能超过10000字"
        }
    ]
}
```

### 2.5 分页参数规范

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码（从1开始） |
| page_size | int | 10 | 每页数量（最大100） |
| ordering | string | -created_at | 排序字段（-表示倒序） |

### 2.6 过滤参数规范

| 参数类型 | 格式 | 示例 |
|----------|------|------|
| 精确匹配 | `field=value` | `status=1` |
| 多值匹配 | `field=value1,value2` | `status=1,2` |
| 范围查询 | `field__gte=value` / `field__lte=value` | `created_at__gte=2026-01-01` |
| 搜索查询 | `search=keyword` | `search=Vue` |

## 3. 认证与授权

### 3.1 JWT认证流程

```
┌──────────┐         ┌──────────┐         ┌──────────┐
│   前端   │         │   后端   │         │   Redis  │
└─────┬────┘         └─────┬────┘         └─────┬────┘
      │                    │                    │
      │  1. POST /auth/login                   │
      │  {username, password}                  │
      │───────────────────►│                    │
      │                    │                    │
      │                    │  2. 验证用户        │
      │                    │───────────────────►│
      │                    │                    │
      │                    │  3. 生成JWT        │
      │                    │◄───────────────────│
      │                    │                    │
      │  4. 返回 {access_token, refresh_token} │
      │◄───────────────────│                    │
      │                    │                    │
      │  5. 请求携带 Authorization: Bearer xxx │
      │───────────────────►│                    │
      │                    │                    │
      │                    │  6. 验证JWT        │
      │                    │───────────────────►│
      │                    │                    │
      │                    │◄───────────────────│
      │                    │                    │
      │  7. 返回数据        │                    │
      │◄───────────────────│                    │
      │                    │                    │
```

### 3.2 Token说明

| Token类型 | 有效期 | 存储位置 | 用途 |
|-----------|--------|----------|------|
| access_token | 2小时 | 前端localStorage | 访问API资源 |
| refresh_token | 7天 | Redis | 刷新access_token |

### 3.3 权限等级

| 权限等级 | 说明 | 可访问资源 |
|----------|------|------------|
| Anonymous | 未登录访客 | 公开的GET接口 |
| User | 已登录用户 | 公开接口 + 评论、点赞、收藏 |
| Admin | 管理员 | 所有接口 + 内容管理 |

## 4. 接口模块详细设计

### 4.1 认证模块 (Authentication)

#### 4.1.1 用户登录

**接口地址**：`POST /api/v1/auth/login`

**请求参数**：
```json
{
    "username": "admin",
    "password": "password123"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名（或邮箱） |
| password | string | 是 | 密码 |

**响应数据**：
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "access_token": "eyJhbGciOiJIUzI1NiIs...",
        "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
        "expires_in": 7200,
        "user": {
            "id": 1,
            "username": "admin",
            "nickname": "管理员",
            "avatar": "https://example.com/avatar.jpg",
            "role": 1
        }
    }
}
```

#### 4.1.2 用户注册

**接口地址**：`POST /api/v1/auth/register`

**请求参数**：
```json
{
    "username": "newuser",
    "email": "user@example.com",
    "password": "password123",
    "password_confirm": "password123"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名（4-20字符） |
| email | string | 是 | 邮箱地址 |
| password | string | 是 | 密码（6-20字符） |
| password_confirm | string | 是 | 确认密码 |

**响应数据**：
```json
{
    "code": 200,
    "message": "注册成功",
    "data": {
        "user_id": 2,
        "username": "newuser"
    }
}
```

#### 4.1.3 刷新Token

**接口地址**：`POST /api/v1/auth/refresh`

**请求参数**：
```json
{
    "refresh_token": "eyJhbGciOiJIUzI1NiIs..."
}
```

**响应数据**：
```json
{
    "code": 200,
    "message": "刷新成功",
    "data": {
        "access_token": "eyJhbGciOiJIUzI1NiIs...",
        "expires_in": 7200
    }
}
```

#### 4.1.4 用户注销

**接口地址**：`POST /api/v1/auth/logout`

**认证要求**：需要JWT认证

**响应数据**：
```json
{
    "code": 200,
    "message": "注销成功"
}
```

#### 4.1.5 获取当前用户信息

**接口地址**：`GET /api/v1/auth/me`

**认证要求**：需要JWT认证

**响应数据**：
```json

    "code": 200,
    "message": "success",
    "data": {
        "id": 1,
        "username": "admin",
        "email": "admin@example.com",
```

## 6. 接口限流策略

### 6.1 限流规则

| 接口类型 | 限流规则 | 说明 |
|----------|----------|------|
| 公开GET接口 | 60次/分钟 | 普通访问 |
| 认证GET接口 | 120次/分钟 | 登录用户 |
| POST接口 | 30次/分钟 | 创建操作 |
| 上传接口 | 10次/分钟 | 文件上传 |
| 登录接口 | 5次/分钟 | 防暴力破解 |
| 评论接口 | 5次/分钟 | 防刷评论 |

### 6.2 限流响应

当触发限流时，返回：
```json
{
    "code": 429,
    "message": "请求过于频繁，请稍后再试",
    "data": {
        "retry_after": 60
    }
}
```

### 8.2 Serializer命名规范

| Serializer类型 | 命名规范 | 用途 |
|----------------|----------|------|
| ListSerializer | `{Model}ListSerializer` | 列表展示 |
| DetailSerializer | `{Model}DetailSerializer` | 详情展示 |

| UpdateSerializer | `{Model}UpdateSerializer` | 更新请求 |

### 8.3 ViewSet使用建议

```python
from rest_framework import viewsets

class ArticleViewSet(viewsets.ModelViewSet):
    queryset = Article.objects.all()
    serializer_class = ArticleListSerializer
    
    def get_serializer_class(self):
        if self.action == 'list':
            return ArticleListSerializer
        elif self.action == 'retrieve':
            return ArticleDetailSerializer
        elif self.action == 'create':
            return ArticleCreateSerializer
        return ArticleUpdateSerializer
```
