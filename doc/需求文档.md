# 学习成果展示网站需求文档

## 1. 项目概述

### 1.1 项目背景

构建一个个人学习成果展示网站，用于记录学习历程**、**展示项目作品、分享技术知识，同时作为求职时的能力展示平台。

### 1.2 项目目标

- 记录个人学习历程，形成知识体系
- 展示项目作品和技术能力，辅助求职
- 分享知识经验，建立个人品牌影响力
- 提供良好的用户体验和访问性能

### 1.3 目标受众

- **个人**：作为学习记录和成长档案
- **招聘方**：了解技术能力和项目经验
- **学习者社区**：获取知识分享和技术交流

***

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 首页展示

- 个人简介与头像展示
- 核心技能标签云
- 最新作品/文章推荐
- 学习数据统计（文章数、项目数、访问量等）

#### 2.1.2 项目作品展示

- 项目列表（支持分类、标签筛选）
- 项目详情页
  - 项目名称、简介、封面图
  - 技术栈标签
  - 项目截图/演示视频
  - GitHub链接、在线演示地址
  - 项目背景与实现难点说明
- 项目点赞、收藏功能

#### 2.1.3 技术文章/博客

- 文章列表（支持分类、标签、时间归档）
- 文章详情页
  - Markdown渲染支持
  - 代码高亮显示
  - 目录导航
  - 阅读时间估算
- 文章评论功能
- 文章点赞、收藏功能

#### 2.1.4 学习笔记

- 笔记分类管理（按技术领域/学习阶段）
- 笔记内容展示
- 学习路线图展示
- 学习进度追踪

#### 2.1.5 关于我

- 个人详细信息
- 教育背景
- 工作经历
- 技能图谱
- 联系方式

### 2.2 用户交互功能

#### 2.2.1 访客功能

- 浏览所有公开内容
- 文章评论
- 项目/文章点赞、收藏
- 搜索功能（全文搜索）
- RSS订阅

#### 2.2.2 管理员功能（本人使用）

- 内容管理（增删改查）
- 项目管理
- 文章/笔记管理
- 评论审核与管理
- 数据统计分析
  - 访问量统计
  - 热门内容分析
  - 访客来源分析

### 2.3 其他功能

- 响应式设计（PC端、移动端适配）
- 深色/浅色主题切换
- SEO优化
- 网站地图生成
- 社交媒体分享

***

## 3. 非功能性需求

### 3.1 性能需求

| 指标      | 要求          |
| ------- | ----------- |
| 首页加载时间  | < 2秒        |
| API响应时间 | < 500ms     |
| 并发用户数   | 支持100+并发访问  |
| 静态资源缓存  | 利用CDN和浏览器缓存 |

### 3.2 安全需求

- XSS防护
- CSRF防护
- SQL注入防护
- 接口限流（防止恶意请求）
- 敏感信息加密存储
- HTTPS支持

### 3.3 可维护性需求

- 代码结构清晰，模块化设计
- 完善的API文档
- 单元测试覆盖率 > 60%
- 日志记录与监控

### 3.4 可扩展性需求

- 支持后续功能模块扩展
- 数据库设计预留扩展字段
- 支持微服务架构演进

***

## 4. 技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户端                                │
│              (PC浏览器 / 移动浏览器)                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     Nginx (负载均衡)                         │
│           - 静态资源服务                                      │
│           - 反向代理                                         │
│           - 负载均衡                                          │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│      前端服务            │     │      后端API服务        │
│        Vue 3            │     │       Django            │
│                         │     │                         │
│  - 页面渲染              │     │  - 业务逻辑处理          │
│  - 路由管理              │     │  - RESTful接口          │
│  - 状态管理(Pinia)       │     │  - 权限验证            │
└─────────────────────────┘     └─────────────────────────┘
                                          │
                              ┌───────────┴───────────┐
                              ▼                       ▼
                    ┌─────────────────┐    ┌─────────────────┐
                    │  Redis 缓存      │    │   MySQL 数据库  │
                    │                 │    │                 │
                    │ - 会话缓存       │    │ - 用户数据      │
                    │ - 热点数据缓存   │    │ - 内容数据      │
                    │ - 访问计数       │    │ - 日志数据      │
                    └─────────────────┘    └─────────────────┘
```

### 4.3 数据库设计概要

#### 主要数据表

1. **users** - 用户信息表
2. **projects** - 项目作品表
3. **articles** - 文章表
4. **notes** - 学习笔记表
5. **categories** - 分类表
6. **tags** - 标签表
7. **comments** - 评论表
8. **likes** - 点赞记录表
9. **favorites** - 收藏记录表
10. **visit\_logs** - 访问日志表

## 5. 部署架构

### 5.1 Docker容器规划

```yaml
services:
  - nginx          # 反向代理
  - frontend       # 前端应用
  - backend        # 后端API
  - mysql          # 数据库
  - redis          # 缓存服务
```

### 5.2 部署流程

1. 代码提交至Git仓库
2. CI/CD自动构建镜像
3. 推送镜像至镜像仓库
4. 服务器拉取最新镜像
5. Docker Compose编排启动
6. Nginx负载均衡配置

### 5.3 环境规划

- **开发环境**：本地开发调试
- **测试环境**：功能测试验证
- **生产环境**：正式对外服务

- 首页加载时间 < 2秒
- API平均响应时间 < 500ms
- 支持100+并发用户访问

### 7.3 安全验收

- 通过基础安全扫描
- 无明显安全漏洞
- 敏感数据加密存储
